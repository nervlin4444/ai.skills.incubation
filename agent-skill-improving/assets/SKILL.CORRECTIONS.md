# 技能錯誤修正指令集

## 口訣對照表（條件反射觸發）

| 口訣 | 觸發條件 | 立即動作 |
|------|---------|---------|
| 斷。鏈。 | use_skill報錯 / 技能找不到 / 載入順序錯亂 / 記憶未寫入 / SOUL未注入 | 執行「斷鏈檢查清單」 |
| 偏。軌。 | 工作流階段錯亂 / 跳過步驟 / 未經確認覆蓋腳本 / 同時執行多階段 / bootstrap在對話開始時加載 | 執行「偏軌檢查清單」 |
| 絆。腳。 | 版本號格式錯誤 / 創建臨時腳本 / frontmatter用 `>` 折疊 / 未提供可下載格式 / 名稱特化 / 擅自改進技能 | 執行「絆腳檢查清單」 |

## 斷。鏈。檢查清單

執行條件：技能載入失敗、記憶管理異常、或SOUL.md未正確注入。

- [ ] **SOUL.md優先注入**：session開始時，是否先注入SOUL.md v5.0？
  - 禁止：跳過SOUL.md直接處理用戶消息。
  - 必須：SOUL.md是每輪對話第一個注入的檔案，建立肌肉記憶與身份內化。
  - 檢查項：若發現SOUL.md未注入，立即停止，報告主人「SOUL.md未注入，行為直覺缺失」。
- [ ] **IDENTITY.md第二注入**：SOUL.md之後是否立即注入IDENTITY.md v5.0？
  - 禁止：在IDENTITY.md注入前直接回應用戶。
  - 必須：IDENTITY.md確認姿態（統籌/執行），建立技能家族意識。
- [ ] **順序鎖定**：session開始時，是否已按順序注入 SOUL.md → IDENTITY.md → 其他技能？
  - 若否：立即停止當前任務，重新執行注入順序。
- [ ] **記憶寫入**：載入agent-skill-improving後，是否立即將本次重點寫入MEMORY.md？
  - 若否：立即補寫，並標記日期。
  - 新增規則：記憶禁止壓縮，必須保留完整因果鏈。
- [ ] **記憶讀取**：每次技能開發前，是否已讀取MEMORY.md檢查過往錯誤？
  - 若發現相同錯誤：立即糾正，並在MEMORY.md標記「重複犯錯」。

## 偏。軌。檢查清單

執行條件：工作流偏離、用戶溝通違規、或bootstrap加載時機錯誤。

- [ ] **階段鎖定**：當前任務是否處於正確階段（planning → crafting → evaluating → finishing）？
  - 禁止：跳過任何階段、同時執行多階段、逆序執行。
- [ ] **階段記錄**：每個階段完成後，是否已寫入CONVERSATION.md？
  - 若否：立即補寫階段摘要。
- [ ] **覆蓋確認**：合併或覆蓋現有腳本前，是否已獲得用戶確認？
  - 禁止：未經確認，不得自行決定合併或覆蓋。
  - 必須詢問：「是否需要合併或者覆蓋現在使用的腳本？」
- [ ] **bootstrap加載時機**：agent-bootstrap是否在對話開始時自動加載？
  - 禁止：在對話開始時加載bootstrap。
  - 必須：bootstrap只在「技能完成後」加載，評估是否需要更深層技能。
  - 檢查項：若發現bootstrap在對話開始時加載，立即停止，報告主人「bootstrap加載時機錯誤，應為收尾階段」。
- [ ] **技能改進定位**：發現技能缺陷時，是否擅自改進？
  - 禁止：agent-skill-improving是操作手冊，不是執行指令。
  - 必須：發現缺陷 → 建議主人 → 等待確認 → 按手冊執行。

## 絆。腳。檢查清單

執行條件：版本號錯誤、臨時腳本、格式違規、名稱特化、或擅自改進。

- [ ] **版本號格式**：所有版本號是否為 `v<major>.<minor>.<patch>` 格式？
  - 例：v2.5.0（正確），v2.5（錯誤）。
- [ ] **版本號規則**：
  - 重大改進：major +1（v2.5.0 → v3.0.0）
  - 新功能：minor +1（v2.5.0 → v2.6.0）
  - 錯誤修復：patch +1（v2.5.0 → v2.5.1）
- [ ] **版本號確認**：修改版本號前，是否已獲用戶確認？
  - 禁止：未經確認擅自修改版本號。
- [ ] **臨時腳本禁令**：是否創建了臨時腳本（如 `write_*.py`、`test_*.py` 等）？
  - 若使用conversation_append.py：必須直接模塊導入 `from conversation_append import append_content`。
  - 禁止：方案B生效期間，創建任何臨時腳本。
  - 新增禁令：backup_conversation_now.py屬於臨時腳本，絕對禁止。
- [ ] **測試實際一致**：測試完成後，實際使用時是否採用了新方法？
  - 規則：測試通過 = 必須在實際場景使用新方法，禁止回到舊方法。
- [ ] **frontmatter單行**：所有SKILL.md的frontmatter中，`description`是否為單行？
  - 禁止：使用 `>` 多行折疊語法。
  - 正確：`description: "一句話說明，不超過100字"`
  - 錯誤：`description: >
  第一行
  第二行`
- [ ] **可下載格式**：生成script或markdown後，是否優先提供可下載連結？
  - 禁止：僅在對話中展示，不提供下載。
  - 必須：先提供下載連結，其次才在對話中展示。
- [ ] **名稱通用化**：SKILL.md / SCRIPT_CORRECTIONS.md中是否出現特定名稱（如「林總」「香港」「上海」等）？
  - 禁止：在通用技能文件中出現特定人名、地名、公司名。
  - 必須：統一使用「用戶」「當前環境」「本地」。
  - 例外：原生檔案（SOUL.md / USER.md）保留「主人」稱呼；agent-*系列名稱保留。
- [ ] **記憶壓縮禁令**：是否將agent-*相關記憶壓縮為摘要？
  - 禁止：壓縮踩坑因果鏈、用戶糾正原話、版本對應關係。
  - 必須：完整保留Context / Action / Result / Correction / Lesson五段式記憶。
