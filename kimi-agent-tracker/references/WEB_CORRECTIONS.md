Kimi Agent Tracker Web Corrections v1.0.0
=========================================

Document Purpose
----------------

This file records all lessons learned from the 2026-05-20 to 2026-05-28
debugging cycle. It exists to prevent any LLM (including future versions
of this assistant) from repeating the same mistakes.

CRITICAL RULE: Read this file BEFORE writing any code for kimi-agent-tracker.


Section 1: Mandatory Workflow (LOCKED)
--------------------------------------

The following workflow is ABSOLUTE. No deviation allowed.

    Step 1: PLAN
        - User describes the goal
        - LLM asks clarifying questions ONLY
        - LLM does NOT write code at this stage

    Step 2: TRACE EXPERIMENT
        - User runs a minimal manual test with --trace enabled
        - User uploads trace_summary.json (generated by trace_minifier.py)
        - LLM analyzes REAL DOM structure from trace
        - LLM identifies exact selectors from trace data, not from guessing

    Step 3: SCREENSHOT PROOF
        - User provides DevTools screenshot or Copy selector results
        - LLM confirms selector accuracy against trace data
        - Mismatch = stop and re-trace

    Step 4: CODE OUTPUT
        - Only after Step 2 and Step 3 are complete
        - LLM references this CORRECTIONS file for forbidden patterns
        - LLM outputs ONCE, with version bump

    Step 5: VALIDATION
        - User runs the code
        - Success = lock the version
        - Failure = go back to Step 2, NOT Step 4

Violation of this sequence (skipping trace/screenshot and going straight to
code) has caused 7 days of wasted debugging. This is a PERMANENT LOCK.


Section 2: LLM Commands That Are FORBIDDEN (LOCKED)
-------------------------------------------------

The following actions are ABSOLUTELY PROHIBITED for any LLM:

    [FORBIDDEN-001] Output code before trace analysis
        - Past violation: v5.1.0 ~ v5.1.4 generated without trace proof
        - Consequence: All versions failed, wasted 20+ iterations

    [FORBIDDEN-002] Guess selectors from sandbox memory
        - Past violation: "div[class*=\"conversation\"]" guessed wrong
        - Consequence: 30s timeout, zero matches
        - Correct method: Read trace_summary.json DOM classes

    [FORBIDDEN-003] Use page.goto() on sandbox:// links
        - Past violation: v5.0.8 tried to navigate to sandbox:// URLs
        - Consequence: ERR_ABORTED, page crash
        - Correct method: Click the file card element, never navigate

    [FORBIDDEN-004] Use innerText as primary download method
        - Past violation: v5.1.1 ~ v5.1.3 relied on preview panel extraction
        - Consequence: 1KB files (1186 chars), only frontmatter captured
        - Correct method: expect_download() for browser download event

    [FORBIDDEN-005] Use chat_title for filename prefix
        - Past violation: Chinese titles became "________xxx.py"
        - Consequence: Illegal filenames, path errors
        - Correct method: Use chat_id or basename only

    [FORBIDDEN-006] Blind sleep() without timeout/retry
        - Past violation: sleep(4) everywhere, no timeout mechanism
        - Consequence: 10+ minutes per chat, unpredictable
        - Correct method: Playwright wait_for_selector with explicit timeout

    [FORBIDDEN-007] Skip local cache check
        - Past violation: Re-downloaded same files across sessions
        - Consequence: Wasted bandwidth, file collisions
        - Correct method: HEAD pre-check or file existence check before download

    [FORBIDDEN-008] Output code without reading SCRIPT_CORRECTIONS.md
        - Source: agent-skill-improving/SKILL.md LOCK-017
        - Past violation: Generated code with // comments in Python
        - Consequence: SyntaxError, unterminated string literal

    [FORBIDDEN-009] Iterate code without user confirmation
        - Source: agent-skill-improving/SKILL.md LOCK-009~012
        - Past violation: Auto-generated v5.1.1, v5.1.2, v5.1.3 without approval
        - Consequence: User frustration, context explosion

    [FORBIDDEN-010] Use nth-of-type for dynamic lists
        - Past violation: Probe v1.1.0 used :nth-of-type({idx})
        - Consequence: Index misalignment, clicked wrong elements
        - Correct method: Playwright >> nth={idx} syntax

Section 4: File Type Download Behavior (Confirmed)
--------------------------------------------------

    .py / .json / .csv
        Behavior: Direct browser download (expect_download captures)
        Menu: None
        Fallback: content_extraction from pre code
        Verified: 2026-05-27

    .md
        Behavior: Click download button -> format menu appears
        Menu: "Download as PDF", "Download as Word", "Download as Markdown"
        Action: Click "Markdown" option -> then expect_download captures
        Fallback: content_extraction from markdown-container
        Verified: 2026-05-27 via screenshot

    .zip
        Behavior: Direct browser download (binary, no preview)
        Menu: None
        Fallback: None (cannot use innerText for binary)
        Verified: 2026-05-27


Section 5: Memory Points That Are Easily Lost
---------------------------------------------

The following facts exist in LLM memory but are frequently forgotten
during code generation. This section exists as an external anchor.

    [MEMORY-001] HEAD Pre-Check Mechanism
        Source: Memory #48
        Content: Before downloading, send HEAD request to get Content-Length
                  and filename. Compare with local file. Same size = skip.
                  Different size = re-download (new version).
                  SHA256 only used for post-download verification, not pre-check.
        Why forgotten: LLM defaults to naive download-everything approach

    [MEMORY-002] Daemon Timeout Strategy
        Source: Memory #48
        Content: First attempt timeout = 20s. If fails, next daemon trigger
                  increases timeout. Purpose: network may improve, not to
                  download larger files. All files < 1MB anyway.
        Why forgotten: LLM adds sleep(4) everywhere instead of timeout+retry

    [MEMORY-003] Trace Minifier Compression Ratio
        Source: Memory #48
        Content: trace.zip (2MB+) -> trace_minifier.py -> trace_summary.json (30-50KB)
                  Compression ratio: 40-60x
        Why forgotten: LLM asks for raw trace files, causing context explosion

    [MEMORY-004] File Naming Convention
        Source: Memory #4
        Content: All skill files must use xxx.yyy.zzz.ext format (dot-separated)
                 .py scripts are EXEMPT and may use xxx_yyy_zzz.py (underscore)
        Why forgotten: LLM generates kimi_downloader-v513.py (dash) or mixed formats

    [MEMORY-005] Frontmatter Mandatory Fields
        Source: Memory #14-16
        Content: title, name, description, version, github_repository,
                  target_branch, updated_at, auth_config, file_mapping
        Why forgotten: LLM omits updated_at or uses wrong github_repository

    [MEMORY-006] Agent-Skill-Improving LOCK Rules
        Source: Memory #42-44
        Content: LOCK-009~016 absolute prohibitions
                 009: No direct open()/write_text()
                 010: No old script names
                 011: Must run integrity checker after modification
                 012: No github_repository/target_branch modification
                 013: fixes field must exist
                 014: Code fixes must match frontmatter
                 015: No separate frontmatter read/generate
                 016: .py must be ASCII (except frontmatter title/name)
        Why forgotten: LLM generates non-ASCII body text or forgets fixes field

    [MEMORY-007] BrowserContext vs Browser
        Source: Memory #23
        Content: launch_persistent_context() returns BrowserContext, not Browser.
                  BrowserContext has no .contexts attribute.
                  Close with await context.close(), not await browser.close().
        Why forgotten: LLM writes browser.contexts[0] causing AttributeError

    [MEMORY-008] Markdown Formatting Rules
        Source: Memory #2
        Content: Remove all ``` fences. Use 4-space indent or tables.
                 This applies to ALL markdown output including corrections.
        Why forgotten: LLM defaults to ```python blocks in documentation

Section 7: Correct Download Sequence (For Reference)
----------------------------------------------------

This is the ONLY validated sequence for downloading a single file:

    1. Navigate to chat URL
    2. Scroll to load history (trigger lazy-loaded messages)
    3. Scan for files (file cards + sandbox links)
    4. Click file element (card or link)
    5. Wait for preview panel (div.side-console-container)
    6. Click download button (.icon-button.primary)
    7. For .md: wait for format menu, click "Markdown"
    8. Capture with expect_download(timeout=10000)
    9. If expect_download fails: extract content from pre code (fallback)
    10. Close preview panel (Escape or close button)
    11. Proceed to next file

For batch mode (multiple chats), insert between step 11 and step 1:
    - page.goto("about:blank") OR
    - page.reload() OR
    - Verify DOM is clean before next navigation
    (This needs trace evidence to confirm which works.)
